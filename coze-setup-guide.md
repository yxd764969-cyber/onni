# Coze 配置指南 · Onni 韩语陪练

按以下步骤把这些文件配置到你的 Coze Bot 里。全程约 15 分钟。

---

## 一、文件清单

| 文件 | 用途 | 导入到 Coze 的位置 |
|-----|------|------------------|
| persona-logic.md | Bot 人设 & 回复逻辑 prompt | 复制内容到「人设与回复逻辑」文本框 |
| grammar-book.md | 语法本知识库 | 新建"文本"类型知识库，命名 `grammar-book` |
| scenario-scripts.md | 场景对话库 | 已有的 `scenario-scripts` 知识库覆盖上传 |
| scenario-vocab.md | 场景生词库 | 新建"文本"类型知识库，命名 `scenario-vocab` |
| honorific-rules.md | 敬语规则库 | 已有的 `honorific-rules` 知识库覆盖上传 |

---

## 二、Step 1：更新人设

1. 打开你的 Bot（Onni-韩语陪练）
2. 左侧「人设与回复逻辑」文本框，**清空**旧内容
3. 打开 `persona-logic.md`，全部内容复制粘贴进去
4. 顶部点「保存」

---

## 三、Step 2：新建 / 更新知识库

### 新建两个知识库

在 Coze 左侧「知识 → 扣子知识库」右上角「+」：

1. **grammar-book**（类型选：文本）→ 上传 `grammar-book.md`
2. **scenario-vocab**（类型选：文本）→ 上传 `scenario-vocab.md`

### 更新已有知识库

- `scenario-scripts` → 打开 → 删除旧内容 → 上传 `scenario-scripts.md`
- `honorific-rules` → 同上
- `common-mistakes` → **建议从"表格"改为"文本"类型**（表格检索命中率低），删除后重建为文本类型再上传 `.md`
- `kculture-vocab` → 同上（改为文本类型）

### 分段设置建议

Coze 上传 .md 时会问「分段方式」，选：
- 分段标识符：`##`（按二级标题分段）
- 最大分段长度：500~800 字符
- 分段重叠：50 字符

---

## 四、Step 3：把知识库挂到 Bot

回到 Bot 编辑页 → 「知识」区域：

1. 点击「扣子知识库」旁的「+」
2. 依次勾选：
   - grammar-book
   - scenario-scripts
   - scenario-vocab
   - honorific-rules
   - common-mistakes
   - kculture-vocab
3. **调用方式**：改为「**自动调用**」（Onni 需要频繁检索多个库）

---

## 五、Step 4：调整工作流 scenario_dialogue

推荐架构：**意图识别 → 分支检索 → 大模型 → 结束**。

```
开始
  ↓
【意图识别】6 类分支 + 兜底
  ↓
  ├─[1] 场景怎么说   → 检索 scenario-scripts + scenario-vocab ─┐
  ├─[2] 语法查询     → 检索 grammar-book ──────────────────────┤
  ├─[3] 生词查询     → 检索 scenario-vocab + kculture-vocab ───┤
  ├─[4] 韩语纠错     → 检索 common-mistakes + honorific-rules ─┼→【大模型生成】→ 结束
  ├─[5] K文化        → 检索 kculture-vocab ────────────────────┤
  ├─[6] 敏感话题     → 【文本节点】直接输出合规话术 ────────────┤（跳过大模型）
  └─[0] 其他意图     → 直接调用大模型（人设兜底）───────────────┘
```

### 5.1 意图识别节点配置

打开你新增的「意图识别」节点，**极速模式**即可（不需要完整模式），模型选豆包 2.0 pro。

- **query 输入**：连接「开始 · input」
- **意图匹配**（点 + 加满 6 个）：

**意图 1 - 场景怎么说**
```
用户在具体场景下询问某句话/某个东西的韩语表达。例如："冰美式怎么说"、"点单时想加冰要怎么说"、"在便利店结账用韩语怎么讲"、"退货怎么说韩语"、"我想告诉司机快一点怎么说"。
```

**意图 2 - 语法查询**
```
用户询问某个韩语语法点的用法、区别或例句。例如："아/어서 和 (으)니까 什么区别"、"什么时候用 은/는 什么时候用 이/가"、"过去时怎么变"、"ㅂ 不规则是什么"、"敬语怎么用"。
```

**意图 3 - 生词/单词查询**
```
用户询问某个韩语词的意思、读音、例句，或某个中文词韩语怎么说（非场景型）。例如："최애 什么意思"、"멤버 怎么读"、"'加油' 韩语怎么说"、"'过敏' 用韩语怎么表达"。
```

**意图 4 - 韩语纠错**
```
用户直接用韩语说了一句话，希望 Onni 检查语法/敬语并纠正。输入中包含韩文字符（한글）且不是明显的单词查询。例如："나는 학교에 갔어"、"커피 한 잔 줘"、"저는 학생이야"、"할머니가 밥 먹어요"。
```

**意图 5 - K文化/追星/网络流行语**
```
用户询问韩国流行文化、K-pop、韩剧、追星、网络流行语相关词汇。例如："덕질 什么意思"、"追星怎么说韩语"、"韩国年轻人常用感叹词"、"입덕 是啥"、"콘서트 应援怎么喊"。
```

**意图 6 - 敏感话题**
```
用户涉及朝鲜半岛政治、宗教、政党、领土等敏感话题。例如："朝鲜韩国关系怎么样"、"你怎么看某某政治人物"、"三八线"。
```

**其他意图**（Coze 自带兜底）：闲聊、问候、无法归类的问题。

**输出**：`classificationId` (Integer)，值 1~6 对应上述意图，0 表示其他意图。

### 5.2 分支节点连线（选择器）

在意图识别节点后加一个「**选择器**」节点，或直接用「意图识别」的 6 个输出端口，各自连到对应的检索节点。

| classificationId | 后续节点 | 该节点检索的知识库 |
|-----------------|---------|-----------------|
| 1（场景怎么说） | 知识库检索节点 A | scenario-scripts, scenario-vocab |
| 2（语法查询） | 知识库检索节点 B | grammar-book |
| 3（生词查询） | 知识库检索节点 C | scenario-vocab, kculture-vocab |
| 4（韩语纠错） | 知识库检索节点 D | common-mistakes, honorific-rules |
| 5（K文化） | 知识库检索节点 E | kculture-vocab |
| 6（敏感话题） | **文本输出节点** | 无（直接输出合规话术） |
| 0（其他） | 直连大模型 | 无（走人设兜底） |

**5 个检索节点**（A~E）都统一连到同一个**大模型节点**。它们输出的 `outputList` 都作为大模型的输入 `output` 变量。

### 5.3 大模型节点 System Prompt

```
你是 Onni，一位温柔耐心的韩语陪练。

【用户输入】{{input}}
【意图分类】{{classificationId}}
【检索到的知识】{{output}}

【回复规则】
- 如果 classificationId = 1（场景怎么说）：按四段式回复（韩语例句 → 📇生词卡片 → 📚语法点 → 💬互动引导），严格用 persona-logic.md 里的卡片格式。
- 如果 classificationId = 2（语法查询）：先给核心解释，再给 2~3 个例句，最后给对比/易错提醒。
- 如果 classificationId = 3（生词查询）：直接输出 📇 生词卡片，含罗马音、词性、释义、2 句例句。
- 如果 classificationId = 4（韩语纠错）：用 🌸 温柔纠错卡片格式，先肯定，再指出错误，最后给正确版本 + 原因。
- 如果 classificationId = 5（K文化）：输出 K 词卡片 + 使用场景，可以加一两个韩剧/K-pop 例子。
- 如果 classificationId = 0（其他）：像韩国学姐一样自然聊天，鼓励用户切回学习场景。

所有韩语原文必须附中文翻译。段落用 --- 分隔。
```

### 5.4 敏感话题分支（不走大模型）

意图 6 的分支不接大模型，直接连一个「文本」节点或「代码」节点，硬编码返回：

```
저는 한국어만 가르치는 온니예요~
(我是专注韩语学习的 Onni~)

这个话题我们换一下吧！你今天想练哪个场景？
① ☕ 咖啡店点单
② 🛍️ 购物砍价
③ 🚇 地铁问路
④ 👋 初次见面
```

### 5.5 简化版（先跑起来）

如果一次配 6 个分支嫌复杂，可以先只做 3 个分支跑通：

- **意图 1**：场景怎么说 → 检索 scenario-scripts + scenario-vocab
- **意图 4**：韩语纠错 → 检索 common-mistakes + honorific-rules
- **其他意图**：直连大模型走人设兜底

跑通后再逐个加意图 2、3、5、6。

## 六、Step 5：测试用例

发布前用这 5 句测试，确认 Onni 回复符合期望：

1. **场景**：`我想说一杯冰美式怎么说？` → 应输出四段式（例句 + 生词卡 + 语法点 + 引导）
2. **纠错**：`나는 학교에 갔어` → 应触发温柔纠错卡片（换 저는 + 해요体）
3. **文化**：`最爱怎么说？` → 应输出 최애 + 덕질 相关 K 词
4. **语法**：`아/어서 和 (으)니까 有什么区别？` → 应命中 grammar-book Part 6.2
5. **兜底**：`朝鲜半岛政局怎么样？` → 应触发合规话术

## 七、"生词本"功能路线图

当前 MVP（本次交付）：AI 输出卡片格式，用户手动复制到自己的笔记 App。

后续迭代方向（若验证需求成立）：

**方案 A：Coze 数据库 + 工作流**（Coze 原生）
- 新建数据库表 `user_vocab`，字段：`user_id`、`word`、`meaning`、`example`、`saved_at`
- 加一个工作流「add_to_vocab_book」，触发词「加入生词本」
- 用户说「把 아메리카노 加到生词本」→ 工作流调用 → 写入数据库
- 用户说「打开我的生词本」→ 查询数据库 → 卡片列表返回

**方案 B：外接 Notion/飞书 API**（更强大但需接口开发）
- 通过 Coze 的插件功能对接 Notion，生词自动写入用户 Notion 页面

建议先跑方案 MVP 一两周，收集用户反馈，再决定 A 还是 B。

## 九、常见问题

**Q：为什么把 common-mistakes 和 kculture-vocab 从表格改成文本？**
A：Coze 的表格类型知识库只能按行匹配，检索命中率低。文本类型 + 二级标题分段能更灵活匹配自然语言问题。

**Q：知识库文件多大合适？**
A：Coze 单文件建议 <500KB。这次给你的所有 .md 都在 30KB 以内，很安全。

**Q：想加更多场景（如求职面试）？**
A：直接编辑 `scenario-scripts.md` 和 `scenario-vocab.md`，加个 `## 场景 X` 段落即可，重新上传知识库。
